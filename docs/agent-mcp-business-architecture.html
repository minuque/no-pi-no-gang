Agent 运行时 + MCP 业务能力架构

no-pi-no-gang 从 Web 外壳演进为基于 pi SDK 的 智能体编排：pi SDK 负责 智能体循环 与扩展钩子，MCP 负责业务能力协议化，能力网关 负责治理、审计、授权和业务事实源边界。

目标形态：本地优先 + 业务能力可插拔

为什么需要 能力网关

MCP 解决连接协议，不自动解决企业治理。网关 负责把 pi 智能体循环 中的工具请求翻译成受控业务能力， 并统一处理授权、审批、脱敏、限流、审计和幂等。

三类事实源

  • .jsonl 记录会话事实：用户意图、模型输出、工具结果。
  • TaskSession 记录运行事实：状态、队列、取消、重试、重连。
  • 业务数据库记录业务事实：项目、工单、客户、订单等。

业务接入节奏

先只读 资源，再提供建议型 提示，最后开放需要审批的 写入工具。写操作要有 幂等键、 结构化结果和可追溯审计记录。
