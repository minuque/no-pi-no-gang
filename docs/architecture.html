no-pi-no-gang Architecture

pi.dev WebUI — 浏览器 UI · Next.js API · AgentSession 运行态 · ~/.pi/agent/ 事实源

三条主链路

  • • ① 历史读取：Browser → /api/sessions → session-reader → .jsonl
  • • ② 命令发送：Browser → /api/agent → rpc-manager → AgentSession → .jsonl
  • • ③ 事件流：Browser ↔ /api/agent/:id/events ↔ subscribe() (SSE 双向)

模块边界

  • • Browser UI：展示会话、消费 SSE、管理交互状态
  • • Next.js API：校验请求、读取文件、管理 SSE、调用核心模块
  • • session-reader：只读解析历史，不做运行副作用
  • • rpc-manager：维护 AgentSession 生命周期、锁、空闲清理
  • • AgentSession：执行智能体动作、写入会话事实

核心事实源

  • • ~/.pi/agent/sessions/**/*.jsonl — 消息历史、分支树、工具调用
  • • AgentSession（运行时）— 当前智能体活动态、事件流
  • • 本地工作目录 — 文件浏览、上下文定位
  • • models.json / settings.json — 模型供应商、认证、偏好
