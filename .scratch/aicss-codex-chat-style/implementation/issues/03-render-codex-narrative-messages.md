**What to build:** 在 Codex 模式下，以 A「叙事流」呈现真实用户消息和 Agent 文本：内容列居中、用户气泡右对齐、Agent 正文无气泡，并在流式到完成的过程中保持稳定阅读位置。

**Blocked by:** 02 — 提供全局 ChatStyle 切换

**Status:** ready-for-agent

- [ ] Codex 模式按真实事件顺序呈现用户文本与 Agent 文本，不制造活动、引用或结构化内容。
- [ ] 流式正文原位增长并显示光标，完成后移除光标且 Markdown 布局不跳变。
- [ ] 长对话切换与流式更新沿用现有自动滚动、用户锚点、复制、重试和分支导航语义。
- [ ] 内容列使用设计契约中的作用域 token，Claude Renderer 的选择器与视觉不受影响。
- [ ] 基础消息、流式转换、空闲和等待模型状态具有自动化测试，快速验证通过。
